# ArcticTern ATC Setup & Run Guide

This document explains how to set up, run, and verify the Multi-Agent System (MAS) Air Traffic Control simulator.

---

## 1. Setup Instructions

### Prerequisites
- Node.js (v18+)
- npm

### Installation
Clone the repository and install the dependencies:
```bash
npm install
```

### Running Locally
To launch the development server:
```bash
npm run dev
```
Once started, the application will be accessible at: `http://localhost:5173/`

---

## 2. Interactive Simulation Scenarios

The simulator includes four pre-configured scenarios to test the HMAS adaptation capabilities under different conditions:

1. **Normal Operations** (`NORMAL`)
   - Steady, moderate traffic flow (18 flights).
   - Clear weather conditions.
   - Baseline benchmark for scheduling efficiency.

2. **Rush Hour** (`RUSH_HOUR`)
   - High congestion environment (36 flights).
   - Clear weather conditions.
   - Tests runway queues, separation rule compliance, and coordination throughput.

3. **Thunderstorm** (`STORM`)
   - Moderate traffic flow.
   - Active storm cell moving east-to-west across the airport canvas.
   - Forces flight holding, holding patterns, and diversions for low-fuel aircraft.

4. **Runway Closure** (`RUNWAY_CLOSURE`)
   - Moderate-high traffic flow.
   - Central runway `10/28` is closed for maintenance (marked with red 'X' indicators).
   - The HMAS must sequence all arrivals and departures on the remaining three runways.

## 3. UI Controls and Features

- **Play/Pause**: Toggle the core tick loop.
- **Speed Slider**: Run simulation ticks at $1\times$, $5\times$, or $10\times$ speeds.
- **Scenario Selector**: Switch scenarios dynamically; the engine automatically resets schedules, weather states, and runway active indicators.
- **Monospace Thinking Feed**: Inspect real-time decisions, action selections, and Q-values generated by the operational agents.
